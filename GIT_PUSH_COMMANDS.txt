TROUBLESHOOTING

Problem: "Remote already exists"
Solution:
  git remote remove origin
  git remote add origin https://github.com/shaik7867867860089-stack/Child_development_portal.git

Problem: "fatal: could not read Username"
Solution:
  Use GitHub Personal Access Token:
  1. Go to: https://github.com/settings/tokens
  2. Create new token with 'repo' scope
  3. When prompted for password, paste token instead
  
  Or use SSH:
  git remote set-url origin git@github.com:shaik7867867860089-stack/Child_development_portal.git

Problem: "Permission denied (publickey)"
Solution:
  Use HTTPS instead:
  git remote set-url origin https://github.com/shaik7867867860089-stack/Child_development_portal.git

Problem: "branch 'main' set up to track 'origin/main'"
Solution: This is normal! Your branch is now linked to GitHub.

Problem: Large files rejected
Solution:
  Git tracks all files. The .pkl files are large (~3 MB total) but acceptable.
  If rejected, use Git LFS:
  git lfs install
  git lfs track "*.pkl"
  git add .gitattributes
  git add .
  git commit -m "Add ML models with Git LFS"
  git push -u origin main
